\section{Method}
\label{sec:method}
The proposed formulation is organized around a structured interface between scene-level clearance reasoning and contact-feasible multi-robot execution. Given a start--goal query for a large vehicle in a cluttered environment, it first uses a width-aware representation to test whether a $W$-clear corridor already exists. If passage is blocked, the representation returns a small set of reachable \emph{frontier gaps} that define the current bottlenecks for progress. These bottlenecks are then converted into compact multi-robot intervention queries and selectively validated through physics-informed search. The overall pipeline is shown in Fig.~\ref{fig:overall}.

\subsection{Clearance Representation and Frontier Gaps}
\label{subsec:frontier}

We represent the scene with a W-clearance connectivity graph (WCCG), whose role
is not to recover a detailed geometric path at every step, but to expose the
actionable output needed for path clearing. The graph is constructed directly
from obstacle geometry in a width-aware manner. In particular, it uses centroid
nodes to represent obstacle components and bridge relations to represent local
inter-obstacle passages. The resulting graph is
\begin{equation}
\mathcal{G}_W \triangleq (\mathcal{V}, \mathcal{E}_c \cup \mathcal{E}_b),
\end{equation}
where $\mathcal{V}$ contains centroid and bridge nodes, $\mathcal{E}_c$ denotes centroid--bridge
edges, and $\mathcal{E}_b$ denotes bridge--bridge edges. Each bridge edge is annotated
with a local width $w_{uv}$, defined by the distance between the associated
closest points on the two neighboring components. This makes $\mathcal{G}_W$ a compact
width-aware representation of where the scene is already traversable and where
narrow passages arise.

Given the vehicle start and goal, we perform a width-constrained connectivity
query on $\mathcal{G}_W$. If the two lie in the same $W$-clear connected face, then a
feasible corridor already exists and no clearing action is needed. Equivalently,
the WCCG answers the following compact connectivity criterion:
\begin{equation}\label{eq:wccg-criterion}
  \big{(}\mathbf{s}_\texttt{V}^{\texttt{S}},\mathbf{s}_\texttt{V}^{\texttt{G}}
  \; \text{belong to the same face of}\; \mathcal{G}_W\big{)},
\end{equation}
If this criterion fails, the query returns the current reachable frontier loop
$\mathcal{L}$ that separates the two sides under the width constraint. The
bridge--bridge edges visible on $\mathcal{L}$ define the first-hop set of
reachable bottlenecks,
\begin{equation}
\Gamma_{\mathcal{L}} \triangleq \{g_1, \ldots, g_K\},
\end{equation}
where each $g_k$ is a candidate frontier gap exposed by the current clearance
query.

These frontier gaps are the key interface produced by the representation. They
are not arbitrary narrow passages or generic local heuristics, but the first
reachable bottlenecks identified by the width-aware connectivity test. As a
result, subsequent stages do not generate contact actions over the full scene.
Instead, they operate only on $\Gamma_{\mathcal{L}}$, converting these
scene-level clearance bottlenecks into compact contact-feasible intervention
queries for multi-robot pushing.

\subsection{Contact-Feasible Pushing Candidates}
\label{subsec:modes}

For a frontier gap $g \in \Gamma_L$ under consideration, the next question is
whether this reachable bottleneck can be enlarged through executable
multi-robot interaction. The role of this stage is therefore not to solve a
fully general manipulation problem, but to convert a scene-level clearance
bottleneck into a small set of \emph{contact-feasible intervention queries}.

Let $\Omega_m$ denote one of the movable obstacles adjacent to $g$. For each
such obstacle, we sample a small set of short-horizon obstacle motion intents
$\{\mathbf{v}_m^k\}_{k=1}^{K_m}$, where
$\mathbf{v}_m^k \triangleq (v_x,v_y,\omega)\in\mathbb{R}^3$, biased toward
local gap-opening directions while allowing a small number of rotational
components to account for local geometry. These sampled directions define the
candidate pushing intents for the obstacles adjacent to $g$.

For each sampled direction $\mathbf{v}_m^k$, we then generate an associated
pushing mode $\boldsymbol{\xi}\triangleq(\mathcal{C},\mathbf{u})$, where
$\mathcal{C}$ denotes the robot--object contact configuration and
$\mathbf{u}$ the associated contact wrench parameters. Let
$\mathbf{v}_m^{k,\mathrm{B}}\in\mathbb{R}^3$ denote $\mathbf{v}_m^k$
expressed in the body frame of $\Omega_m$. The quality of a candidate mode is
measured by the primary and multi-directional feasibility losses
\begin{equation}
\label{eq:mode-feasibility}
\begin{aligned}
\mathrm{J}_{\texttt{F}}^m
(\boldsymbol{\xi},\mathbf{v}_m^{k,\mathrm{B}})
&\triangleq
\underset{\mathbf{q}_{m,\boldsymbol{\xi}} \in
\mathrm{Q}_{m,\boldsymbol{\xi}}}{\textbf{min}}
\left\|
\mathbf{q}_{m,\boldsymbol{\xi}} +
Q_\mu^m(\mathbf{v}_m^{k,\mathrm{B}})
\right\|_1,\\
\mathrm{J}_{\texttt{MF}}^m
(\boldsymbol{\xi},\mathbf{v}_m^{k,\mathrm{B}})
&\triangleq
\sum_{\mathbf{v}_d \in \mathcal{D}} w_d \cdot
\mathrm{J}_{\texttt{F}}^m(\boldsymbol{\xi},\mathbf{v}_d),
\end{aligned}
\end{equation}
where $\mathbf{q}_{m,\boldsymbol{\xi}} \in \mathbb{R}^3$ is the body wrench
induced by mode $\boldsymbol{\xi}$,
$\mathrm{Q}_{m,\boldsymbol{\xi}} \subseteq \mathbb{R}^3$ is the corresponding
admissible wrench set under local contact and friction constraints,
$Q_\mu^m(\mathbf{v}_m^{k,\mathrm{B}})$ is the quasi-static friction wrench of
$\Omega_m$ along $\mathbf{v}_m^{k,\mathrm{B}}$, and
$\mathcal{D}$ with weights $w_d$ augments the desired direction by a small set
of neighboring basis velocities. Small $\mathrm{J}_{\texttt{F}}^m$ is
necessary for force feasibility, while small $\mathrm{J}_{\texttt{MF}}^m$
indicates a more robust mode. Accordingly, the associated mode
$\boldsymbol{\xi}_m^k \triangleq (\mathcal{C}_m^k,\mathbf{u}_m^k)$ is obtained
by sparse constrained optimization over reachable contacts on
$\partial\Omega_m$, minimizing
$\mathrm{J}_{\texttt{MF}}^m(\boldsymbol{\xi},\mathbf{v}_m^{k,\mathrm{B}})$
under local friction and wrench constraints. Only the best few pairs
$(\mathbf{v}_m^k,\boldsymbol{\xi}_m^k)$ are retained, while a persistent
\emph{ModeTable} reuses effective entries across similar local geometries.
Each retained candidate therefore takes the form
$(g,\mathbf{v},\boldsymbol{\xi})$, and the gap-induced candidate set is
denoted by $\Xi_g$.

A candidate is retained only if it passes short-horizon physical validation.
Specifically, the robots must be able to reach the proposed pre-contact poses,
avoid collision during approach and pushing, maintain stable contact, and
induce obstacle motion that actually enlarges the target gap. This validation is
performed through a short-horizon rollout,
\begin{equation}
\label{eq:sim}
(\mathbf{s}', \delta_{\texttt{T}}, \delta_{\texttt{J}})
\triangleq
\mathsf{EvalSim}\bigl(\mathbf{s}, (g,\mathbf{v},\boldsymbol{\xi})\bigr),
\end{equation}
which returns the successor state, time cost, and control effort. As a result,
this stage does not enumerate contact actions over the full scene; it converts
the reachable frontier gap into a compact set of physically testable
multi-robot interventions for the hybrid search.

\subsection{Selective Validation in Hybrid Search}
\label{subsec:pihs}
At node $\nu$, the method first checks whether the current state satisfies the
connectivity criterion in~\eqref{eq:wccg-criterion}. If not, the updated WCCG
induces a reachable frontier loop $\mathcal{L}_\nu$ and the associated
candidate gaps $\Gamma_{\mathcal{L}_\nu}$. As illustrated in
Fig.~\ref{fig:presearch}, these gaps are ranked by the predicted
cost-to-connect
\begin{equation}
\label{eq:gap-score}
\widehat{\mathsf{Cost}}(g)
\triangleq
\mathsf{C}(g \mid \mathcal{L}_\nu,\mathbf{s}_{\mathcal{R}})
+
\mathsf{C}_{\mathrm{pre}}(g),
\end{equation}
where $\mathsf{C}(g \mid \mathcal{L}_\nu,\mathbf{s}_{\mathcal{R}})$ combines
the robots' transition cost to the gap and the estimated effort required to
widen it, and $\mathsf{C}_{\mathrm{pre}}(g)$ is a short presearch estimate of
downstream gap-crossing cost toward the goal. The ranked list is
\begin{equation}
\label{eq:gap-rank}
\mathsf{Rank}(\nu)
\triangleq
\operatorname{argsort}_{g \in \Gamma_{\mathcal{L}_\nu}}
\widehat{\mathsf{Cost}}(g),
\end{equation}
and the node priority is
\begin{equation}
\label{eq:node-priority}
f(\nu)
=
\chi(\nu)
+
\min_{g \in \mathsf{Rank}(\nu)} \widehat{\mathsf{Cost}}(g),
\end{equation}
which balances realized cost and predicted remaining effort.
Algorithm~\ref{alg:pihs} summarizes the resulting selective expansion loop.

%==============================================
\begin{algorithm}[t!]
\caption{Selective Validation in Physics-Informed Hybrid Search}
\label{alg:pihs}
\SetAlgoLined
\KwIn{$\mathbf{s}_0$, $\alpha$, $\mathsf{EvalSim}(\cdot)$}
\KwOut{$\pi^\star$}
$\nu_0\gets(\mathbf{s}_0,\emptyset),\;\chi(\nu_0)=0$\;
Initialize $\mathcal{Q}$ by~\eqref{eq:node-priority}\;
\While{$\mathcal{Q}\neq\emptyset$}{
  $\nu=(\mathbf{s},\pi)\gets\mathcal{Q}.\texttt{pop\_min}()$\;
  \If{$\mathbf{s}$ satisfies~\eqref{eq:wccg-criterion}}{
    $\pi^\star\gets\pi$, \textbf{break}\;
  }
  Update the WCCG and construct $\mathsf{Rank}(\nu)$\;
  \ForEach{$g\in\mathsf{Rank}(\nu)$}{
    Generate $\Xi_g$\;
    \ForEach{$(\mathbf{v},\boldsymbol{\xi})\in\Xi_g$}{
      $\tau\gets(g,\mathbf{v},\boldsymbol{\xi})$\;
      $(\mathbf{s}',\delta_{\texttt{T}},\delta_{\texttt{J}})
      \gets\mathsf{EvalSim}(\mathbf{s},\tau)$\;
      \If{success}{
        $\nu'\gets(\mathbf{s}',\pi\cup\{\tau\})$\;
        $\chi(\nu')\gets\chi(\nu)+\delta_{\texttt{T}}+\alpha\,\delta_{\texttt{J}}$\;
        $\mathcal{Q}.\texttt{push}(\nu')$\;
      }
    }
  }
}
\Return{$\pi^\star$}\;
\end{algorithm}
%==============================================

For each ranked gap $g \in \mathsf{Rank}(\nu)$, the search calls the
gap-conditioned candidate generator in Sec.~\ref{subsec:modes}. There,
constrained mode generation guided by~\eqref{eq:mode-feasibility} produces a
compact candidate set $\Xi_g$, and each candidate
$\tau \triangleq (g,\mathbf{v},\boldsymbol{\xi})$ is evaluated by the
short-horizon rollout in~\eqref{eq:sim}. PIHS is therefore selective in three
senses: the WCCG restricts attention to reachable bottlenecks, gap ranking
prioritizes which bottlenecks to test first, and candidate generation together
with short-horizon simulation validates only those interventions worth
expanding. Repeating this loop yields a corridor-construction schedule that
specifies which gaps to open and how to open them.
